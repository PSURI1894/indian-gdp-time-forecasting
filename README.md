# Time Series Forecasting — India GDP (a learn-by-building curriculum)

A progressive, project-based course in time-series forecasting, taught on **real
Indian GDP data**. Each project is self-contained (notebooks + a deep-dive doc)
and builds on a shared, reusable toolkit in [`src/`](src/). The goal is depth:
not "import a library and call `.fit()`", but understanding *why* each step works
so it transfers to real forecasting work.

## The curriculum

| # | Folder | Focus | Status |
|---|--------|-------|--------|
| **P1** | [`p1_foundations/`](p1_foundations/) | Foundations & classical: transforms, decomposition, stationarity, baselines, **honest evaluation**, ETS, ARIMA/SARIMA, Theta, Prophet, ARIMAX | ✅ complete (9 notebooks: 00–08) |
| **P2** | `p2_ml/` | Feature-based ML: lag/calendar/Fourier features, gradient boosting, direct vs recursive multi-step | ⬜ planned |
| **P3** | `p3_probabilistic/` | Prediction intervals, quantile loss, conformal prediction | ⬜ planned |
| **P4** | `p4_deep_learning/` | DL forecasting **from scratch in NumPy** (PyTorch is blocked by Smart App Control here) | ⬜ planned |
| **P5** | `p5_capstone/` | End-to-end pipeline + model selection + report | ⬜ planned |

## The data (all real, all reproducible)

Fetched and cached by [`src/data.py`](src/data.py) — no API keys needed.

| series | source | span | role |
|--------|--------|------|------|
| annual real GDP (constant LCU) | World Bank `NY.GDP.MKTP.KN` | 1960–2024 (65) | trend, log-transform, non-seasonal ARIMA, tiny-sample backtests |
| quarterly real GDP, **NSA** | OECD QNA (via DBnomics) | 2004Q2–2026Q1 (88) | seasonality, decomposition, SARIMA |
| quarterly real GDP, SA | OECD QNA (via DBnomics) | 2004Q2–2026Q1 (88) | contrast — what seasonal adjustment removes |

> ⚠️ The three series use different base years / price bases, so their *levels*
> are **not** cross-comparable. Forecasting works within a series, so that's fine.

## Setup

```powershell
# from the repo root
uv venv --python 3.12 .venv
uv pip install -r requirements.txt
.venv\Scripts\python.exe -m ipykernel install --user --name tsf --display-name "Python (TSF)"

# fetch + cache the data
.venv\Scripts\python.exe -m src.data
```

Then open any notebook and select the **Python (TSF)** kernel.

## Repo layout

```
src/                shared toolkit (reused by every project)
  data.py           fetch + cache the 3 GDP series; load_annual(), load_quarterly()
  metrics.py        MAE/RMSE/MAPE/sMAPE/MASE  (MASE is the headline metric)
  backtest.py       rolling-origin (time-series CV) — the leak-free evaluator
  baselines.py      naive / seasonal-naive / drift / mean benchmarks
  plots.py          thin matplotlib helpers
data/               raw/ (source snapshots) + processed/ (tidy CSVs)
docs/               CODE_WALKTHROUGH.md — line-by-line tour of the src/ toolkit
p1_foundations/     Project 1: notebooks/ + build_p1.py + DEEP_DIVE.md (the math)
reports/figures/    saved plots
```

## Docs

- [`docs/CODE_WALKTHROUGH.md`](docs/CODE_WALKTHROUGH.md) — every `src/` module
  explained block-by-block, with the *justification* for each design choice.
- [`p1_foundations/DEEP_DIVE.md`](p1_foundations/DEEP_DIVE.md) — the math behind
  log-transforms, decomposition, stationarity, ETS/ARIMA, and MASE.

Each project's notebooks are generated by a `build_*.py` script (the teaching text
lives in the script, under version control); execute them with:

```powershell
.venv\Scripts\python.exe -m jupyter nbconvert --to notebook --execute --inplace `
  --ExecutePreprocessor.kernel_name=tsf p1_foundations/notebooks/*.ipynb
```
